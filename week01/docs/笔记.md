## 一、Maven 环境配置

### 1. 下载与安装

- 下载渠道：Apache Maven 官网获取二进制压缩包，避免下载源码包。
- 解压要求：将压缩包解压到**非中文、无空格**的目录（如 D 盘 dev 文件夹下），路径混乱会导致后续配置失败。

### 2. 环境变量配置（Windows 系统）

1. 新增系统变量 `MAVEN_HOME`，值为 Maven 解压的根目录。
2. 编辑系统变量 `Path`，添加 `%MAVEN_HOME%\bin`，让系统能识别 Maven 命令。
3. 验证方式：打开命令提示符，输入 `mvn -v`，显示 Maven 版本信息即配置成功。

### 3. 核心优化配置

1. 找到 Maven 解压目录下的 `conf/settings.xml` 文件，用文本编辑器打开。
2. 配置本地仓库：指定一个自定义路径（如 D:\dev\maven-repo），用于存储下载的依赖包，避免默认路径混乱。
3. 配置镜像源：添加阿里云镜像，替换默认的中央仓库，大幅提升依赖下载速度。

## 二、Git 环境配置

### 1. 下载与安装

- 下载渠道：Git 官网下载对应系统版本，安装时建议勾选 “Git Bash Here”，方便右键快速打开命令窗口。
- 验证方式：打开 Git Bash，输入 `git --version`，显示版本号即安装成功。

### 2. 基础身份配置

- 打开 Git Bash，执行命令配置全局用户名和邮箱（需与 Git 账号一致），这是提交代码的身份标识。
- 可通过 `git config --list` 查看已配置的信息，确认无误即可。

### 3. 核心操作思路（无命令）

- 初始化本地仓库：将本地项目与 Git 关联。
- 暂存与提交：把项目文件提交到本地仓库，每次提交需填写备注说明修改内容。
- 关联远程仓库：将本地仓库与 Gitee/GitHub 等远程仓库绑定，实现代码同步。

## 三、IDEA 中配置 SpringBoot 并实现 Hello 接口

### 1. IDEA 关联本地 Maven

1. 打开 IDEA 的设置界面（File > Settings），找到 Maven 配置项。

2. 关键配置：
   
   - 指定本地 Maven 的安装目录，而非 IDEA 默认的内置 Maven。
   - 关联修改后的 settings.xml 文件，确保镜像和本地仓库配置生效。
   - 确认本地仓库路径与 settings.xml 中配置的一致。

3. 保存配置后，IDEA 会按照自定义的 Maven 配置下载依赖。

### 2. 创建 SpringBoot 项目

1. 新建项目时选择 Spring Initializr，确认本地 JDK 版本（建议 1.8 及以上）。
2. 配置项目基本信息：填写组织标识、项目名，确保 Java 版本与本地 JDK 匹配。
3. 选择核心依赖：勾选 “Spring Web”（用于开发 Web 接口），其他默认即可。
4. 等待项目加载：首次创建会下载 SpringBoot 相关依赖，需耐心等待，依赖下载完成后项目结构才完整。

### 3. 编写 Hello 接口的核心步骤

1. 项目结构梳理：
   
   - 启动类：自动生成，是项目运行的入口，无需修改。
   - 控制器目录：手动创建，用于存放接口相关的类。

2. 接口编写思路：
   
   - 新建控制器类，添加注解标识这是一个 Web 接口类（可直接返回数据）。
   - 为类指定统一的访问路径，为接口方法指定具体的请求方式（如 GET）和子路径。
   - 方法返回指定的字符串（如 “Hello SpringBoot!”）。

### 4. 启动与测试接口

1. 启动项目：运行自动生成的启动类，控制台显示项目启动成功的提示（包含启动耗时）即可。

2. 测试方式：
   
   - 浏览器访问：输入 “[localhost:8080](https://localhost:8080) + 类路径 + 方法路径”，页面显示指定字符串即成功。
   - 工具测试：用 Postman 等接口测试工具发送对应请求，查看响应结果。

### 

### 总结

1. **环境配置关键**：Maven 重点配置环境变量和镜像源，Git 重点配置用户身份，IDEA 需正确关联本地 Maven。
2. **SpringBoot 项目核心**：创建时勾选 “Spring Web” 依赖，接口类需添加专属注解，指定访问路径。
3. **接口测试逻辑**：启动项目后，通过 “[localhost:8080](https://localhost:8080) + 自定义路径” 访问，返回预期字符串即实现成功。
